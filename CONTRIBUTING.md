# Contributing to VulnShop-Lab

Thank you for your interest in improving VulnShop-Lab.

## Contribution Ideas

You can contribute by adding:

- New beginner-friendly vulnerabilities
- Secure implementation examples
- More challenges
- Better documentation
- Tests for security behavior
- Report generator templates

## Adding a New Vulnerability

Please include:

1. A clear vulnerability ID, for example `VULN-08`.
2. Vulnerable implementation in `vulnerable-app/`.
3. Secure implementation in `secure-app/`.
4. Documentation in both `docs/vi/` and `docs/en/`.
5. A sample evidence JSON file.
6. A behavioral test if possible.

## Documentation Style

Keep explanations beginner-friendly, evidence-based, and practical. Avoid encouraging testing against unauthorized real-world systems.
