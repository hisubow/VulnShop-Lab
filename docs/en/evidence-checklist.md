# Evidence Checklist - English

When documenting a finding, collect the following information:

- [ ] Vulnerability title
- [ ] Affected endpoint
- [ ] User role/account used for testing
- [ ] Vulnerable app behavior
- [ ] Secure app behavior
- [ ] HTTP status code if available
- [ ] Screenshot or written evidence
- [ ] Root cause in code
- [ ] Security impact
- [ ] Remediation
- [ ] Retest result

The goal is to convert web actions into clear evidence that can be used in a report.
