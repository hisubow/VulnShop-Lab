# Evidence Checklist - Tiếng Việt

Khi ghi nhận một finding, hãy thu thập đủ các thông tin sau:

- [ ] Tên lỗ hổng
- [ ] Endpoint bị ảnh hưởng
- [ ] Vai trò/tài khoản dùng để test
- [ ] Hành vi ở vulnerable app
- [ ] Hành vi ở secure app
- [ ] HTTP status code nếu có
- [ ] Screenshot hoặc mô tả bằng chứng
- [ ] Root cause trong code
- [ ] Impact bảo mật
- [ ] Remediation
- [ ] Retest result

Mục tiêu là biến thao tác web thành bằng chứng rõ ràng, có thể đưa vào report.
